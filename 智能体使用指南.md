# 博客文章智能管理智能体使用指南

## 🎯 智能体概述

这是一个专为AI/ML知识博客设计的智能文章管理智能体，能够自动分类和添加文章到正确的分类目录。

## 🚀 快速开始

### 1. 启动智能体服务

```bash
cd api-server
npm install
npm run dev
```

服务将在 http://localhost:3001 启动

### 2. 测试智能体功能

#### 获取所有可用分类
```bash
curl http://localhost:3001/api/categories
```

#### 智能分类文章
```bash
curl -X POST -H "Content-Type: application/json" -d '{
  "title": "深度学习基础概念",
  "content": "深度学习是机器学习的一个分支..."
}' http://localhost:3001/api/articles/classify
```

#### 从文件自动添加文章
```bash
curl -X POST -H "Content-Type: application/json" -d '{
  "filePath": "C:\\\\Users\\\\Lenovo\\\\Desktop\\\\main\\\\test-article.md",
  "tags": ["深度学习", "神经网络"]
}' http://localhost:3001/api/articles/from-file
```

## 🧠 智能分类能力

### 支持的分类
- **theory** (理论基础) - AI/ML基础理论和概念
- **models** (算法与模型) - 各种机器学习算法和模型
- **practice** (实战与应用) - 实际应用和项目实践
- **tools** (工具与框架) - 开发工具和框架使用
- **insights** (思考与洞见) - 深度思考和行业见解

### 分类机制
1. **AI模型分类** (优先) - 使用HuggingFace API进行智能分类
2. **关键词分类** (备用) - 基于预设关键词匹配，无需外部依赖

## 📁 文件处理

### 自动文件保存
- 文章自动保存到对应的分类目录
- 自动生成符合规范的frontmatter
- 文件名自动slug化处理
- 支持重复文件检测和重命名

### 文件结构
```
src/content/blog/
├── theory/
│   └── 深度学习基础概念.md
├── models/
├── practice/
├── tools/
└── insights/
```

## 🔧 配置选项

### 环境变量
在 `api-server/.env` 文件中配置：

```env
PORT=3001
HUGGINGFACE_API_KEY=your_api_key_here  # 可选，用于AI分类
CONTENT_BASE_PATH=../src/content/blog
```

### 分类关键词定制
可以修改 `src/services/classificationService.ts` 中的关键词配置来优化分类准确性。

## 💡 使用场景

### 批量文章导入
```bash
# 批量处理多个文件
for file in /path/to/articles/*.md; do
  curl -X POST -H "Content-Type: application/json" -d "{
    \"filePath\": \"$file\"
  }" http://localhost:3001/api/articles/from-file
done
```

### 内容管理系统集成
可以与现有的CMS系统集成，通过API自动分类和保存文章。

### 自动化工作流
结合Git hooks或CI/CD流程，实现文章提交时的自动分类。

## 🛠️ 开发扩展

### 添加新的分类
1. 在 `src/content/config.ts` 中添加新的分类枚举
2. 在 `src/types/index.ts` 中更新类型定义
3. 在 `src/services/classificationService.ts` 中添加分类逻辑

### 自定义分类算法
可以替换或扩展 `classificationService.ts` 中的分类逻辑，集成其他AI服务或自定义算法。

## 📊 性能指标

- 分类准确率: > 85% (基于关键词匹配)
- 响应时间: < 500ms (本地处理)
- 支持并发: 多个API请求同时处理

## 🔍 故障排除

### 常见问题
1. **文件路径错误** - 确保使用绝对路径
2. **权限问题** - 确保有文件读写权限
3. **JSON格式错误** - 检查请求体格式

### 日志查看
查看 `api-server/logs/` 目录下的日志文件获取详细错误信息。

## 📝 最佳实践

1. **文件命名**: 使用有意义的文件名，便于识别
2. **内容质量**: 确保文章内容清晰，便于AI准确分类
3. **标签使用**: 合理使用标签提高分类准确性
4. **定期备份**: 定期备份文章内容
